# 简介
这里就是基本的摆放方式了. 整体上来看呢, 稍微有些混乱, 毕竟现在还没有弄复杂的控件. 但是再简单地理解一下就好很多了, 整体上来看, 并不是那么地困难. 

## 控件
### QVBoxLayout
- 实例化
    - 在我看的几个例子, 都是`QV`为主导. 什么意思呢? 就是`QV`传入父类指针, 然后再摆放其它控件或者是`QH`. 
    - 但是, 目前还不确定是否是必须的. 什么意思呢? 就是`QH`为主导, `QH`传入`this`或者父类指针, 然后`QV`当然普通`layouot`, 被放入`QH`中. 目前还不确定. 
- 方法
    - 方法有`addWidget()`, 以及`addLayout()`. 前者就是放入普通控件, 而后者则是放入层级. 目前遇到的后者摆放层级是`QH`类型, 而不是`QV`类型. 当然, 只看了几个例子, 不能一概所有. 
    - 但其实这样就已经解决了我们大部分摆放问题了. 
### QHBoxLayout
- 目前来看, 遇到的`QH`层级, 只用到了`addWidget()` 方法添加控件, 没遇到过添加子层级(用`addLayout`). 
- 但是, 从逻辑上来看, 其实都是可以的. 我觉得是这样的, 虽然我自己还没有验证过.
- 刚才看了看, 其实`QH`和`QV`是完完全全等价的. 所谓的主导, 就只有一个, 而且这个也是完全由你掌控的. 怎么掌控呢? `QWidget`有`setLayout()`方法, 里面传入的谁, 谁就是主导. 
- 为什么我会主观上认为`QV`是主导呢? 因为我遇到的, 几乎都是`setLayout()`内部添加的是`QV`, 而不是`QH`, 这其实就点名了所有问题了, 哈哈. 
- 对了, 其实并不是以`setLayout()`为主导的. 如果, 我们定义的时候, 在构造函数中, 显式地传入了对应指针, 其实就不用`setLayout()`了. 如果没有指定, 可以通过`setLayout()`来后手指定. 
- 可能还存在些许问题, 但目前来看, 已经足够了. 等遇到我现在思考的问题时, 专门分析即可. 
- 对于`addWidget()`方法, 第一个参数是`Widget`对象, 第二个是伸缩因子. 什么意思呢? 如果`add`了两个`widget`, 第一个伸缩因子为`1`, 第二个为`2`, 那么第一个所占空间为整体的`1/3`, 第二个则为`2/3`. 第三个参数是区域中对齐方式. 其中, `AlignCenter` 是水平加垂直居中. 其它的先不用管, 等遇到的时候, 知道即可. 
- `addSpacing()`则是添加空白块, 固定像素的. 
- `addStretch()` 则是添加可伸缩空白, 通常用于居中布局. 怎么说呢, 没怎么用过. 测试练习一下吧, 知道什么用即可. 
- `setAlignment()` 这个也是一个对齐方式. 但是呢, 在`addWidget()`, 也可以选择对齐方式. 这俩有什么区别呢? 区别还是很大的. 前者, 是调控所有的, 在这个`Layout`中的对齐方式, 也就是整体都默认是这里选择的对齐方式. 而通过`addWidget()` 方法, 只是改变这一个控件的对齐方式. 只影响这一个控件, 操控空间十分地大.
