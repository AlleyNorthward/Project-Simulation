# 简介
&emsp;&emsp;思考了很久, 想了想, `Uml`设计图, 还是很有必要的. 结构捋顺不清, 不知道自己要弄什么东西, 就会被搞得非常非常烦. 所以, 对于`Uml`可视化, 理清彼此之间的关系, 还是很有必要的. 现在写代码, 总是一个劲地打算写, 可是总感觉缺少了什么东西. 什么东西呢? 还是心理表征. 说白了, 不会刻意练习. 思考一下, 比如, 我想用`Qt`做一个东西. 可是我并不会`Qt`, 那么应当如何快速学习呢? `Ai`几行代码就能解决我的所有问题吗? 肯定不是的. 比如, 我想弄一个登录窗口, 那么我必须将其所有的依赖类, 都给搞清楚, 运行逻辑搞明白, 才能带着自己的脑子写. 当然, 很多类肯定是无法记住的, 需要的是什么东西呢? 刻意练习, 留下心理表征. 之前说了, 千万不要对着自己的源代码抄, 这种简单的重复, 不会给你带来任何收获. 这时, `Uml` 自然也就登场了. 什么是刻意练习呢? 我觉得是给内心留下深刻心理表征的一种练习, 让我们每每遇到类似情况的时候, 总能第一时间想到自己的心理表征, 从而快速获取答案. `Uml`对于编程而言, 可以说是极为优秀的可视化的工具, 它会给我们留下心理表征吗? 我觉得肯定不会, 但它确实是很好的协助工具. 刻意练习过程中, 我们应当极力避免看着源代码写, 而看着`Uml`图写, 可以当做是一种过度, 最终效果是脱离`Uml`, 完全自主地写出来. `Uml`肯定不是心理表征, 但是, 看着`Uml`图, 思索编写过程时, 或者, 看着`Uml`图, 遇到无法解决问题, 而瞄了一眼答案, 从而恍然大悟时, 心理表征就此落下. 所以, 你说`Uml`存在的意义是什么呢? 构建? 架构? 结构清晰? 我觉得可能因人而异. 再回到`Qt`上来, 它有很多很多东西, 我们可以让`Ai`直接生成, 然而这么做, 能学到什么东西吗? 我觉得需要理清一下. 比如, `Qt`的`Window`, 需要专门研究, `Qt`的摆放逻辑, 专门研究, `Qt`的按键绑定, 专门研究, `Qt`与数据库的交互, 专门研究. 其实, 每个专门研究, 就是一次刻意练习. 而只针对这一小点的话, 问题就会变得十分简单. 可如果我们将各种各样的复杂逻辑放在一块, 整体统一操作的话, 问题就会变得十分棘手. 因为我们并不精通`Qt`, 而是以练习代学习. 一口吃成个胖子? 比如, 对于整个`Window`而言, 内部有很多方法, 我们完全可以专门练习一下, 了解一下基本的方法, 然后按照自己的思想添加一些东西, 这样似乎是更好的选择. 那么`Uml`在这里起到的什么作用呢? 一方面, 是说明这个类中, 包含什么方法, 有什么用处, 另一方面, 也可以在我们抽象函数中, 点明运行逻辑, 这样自己看逻辑, 就知道有什么用处. 当然, 一切都不会很容易, 需要找到一种适合自己学习的方法和路径, 每时每刻吸取教训, 获取经验, 从而不断优化. 好了, 就说这么多吧. 这或许就是我创建这个`Uml`文件夹的意义所在吧. <br>
